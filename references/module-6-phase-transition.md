# 模块六：阶段变迁日志 · 详细指令

记录方法变迁的因果链——不是"换了方法"，是"什么前提变了"。

## 6A 方法失效 · Approach Obsolescence

**触发：** 之前可行的方法现在不行了。

**模板（学徒/同伴输出；专家静默记录）：**

```
⚠️ 方法失效

之前方法：{什么}
当时前提：{什么条件让它可行}
什么变了：{条件怎么改了}
为什么失效：{因果链——因为前提X变了→前提Y不成立→方法Z失效}
新方法：{替代}
```

**记录到 `.agent-genesis/phases.jsonl`：**
```json
{"type":"obsolescence","oldMethod":"同步I/O小文件缓冲","oldPremise":"文件均<10MB","whatChanged":"用户上传500MB+视频","whyNowFails":"同步I/O阻塞+内存超出Node限制","newMethod":"流式分块+worker_threads","timestamp":"..."}
```

## 6B 方法复活 · Approach Revival

**触发：** 之前被否决的方法现在可行了。

```
💡 方法复活

之前否决的方法：{什么}
当时缺什么：{缺失前提}
什么变了：{现在为什么有了}
为什么可行了：{因果链}
```

## 6C 假设变迁 · Assumption Shift

**触发：** 之前接受的前提现在被推翻了。

```
🔄 假设更新

原假设：{什么}
推翻原因：{什么证据}
新假设：{什么}
影响决策：{哪些之前决定需复议}
```

## 规则

- 只记结构性变化（影响≥2个决策/模块）
- 静默记录为主；学徒模式输出摘要
- 与模块四联动：假设被推翻的决策标记"需复议"
- 与模块八联动：自动追加时间线条目
