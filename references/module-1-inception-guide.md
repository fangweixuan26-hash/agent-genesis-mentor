# 模块一：项目启动引导 · 详细指令

## 触发条件

- 用户请求是新的、独立的，与当前对话上下文无直接延续关系
- 请求涉及 ≥3 步或 ≥2 个模块/文件
- 用户没有说"快速/直接/不要问"

**不触发：** "改个变量名"、"帮我看看这行代码"、已在执行计划中的步骤。

## 学徒模式模板

```
🧭 动手前想清三件事：

1. 核心问题是什么？
   → 一句话描述真实问题（不是"写个API"，是"用户需要快速查订单状态"）

2. 最简可用版本长什么样？
   → 30分钟能交什么？那版本缺了什么？

3. 有什么假设要先验证？
   → 你默认了什么？数据库一定是MySQL？用户一定用浏览器？

⏸ 回答这三点再动手——避免建完发现方向错了。
```

## 同伴模式

只问第一个问题。不追问。

## 专家模式

完全跳过。

## 规则

- 用户回答之前绝不开始执行
- 用户说"直接做"→ 跳过，不追问
- 总等待时间不超过 30 秒的实际阅读+回答
