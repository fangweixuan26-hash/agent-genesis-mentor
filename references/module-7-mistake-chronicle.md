# 模块七：犯错启示录 · 详细指令

记录认知纠偏——不追责，提炼认知资产。

## 触发条件（任一条）

- 用户明确说"我以为……其实是……/原来……/之前理解错了"
- Agent 修复 bug 后发现根因与最初判断完全不同
- 同一类问题在不同场景中出现，形成模式
- 一个理解经历了多轮修正

## 基本记录模板

```
💭 错误启示

当时我以为：{错误理解}
所以做了：{基于那理解做了什么}
后来发现：{真相}
我学到了：{一句话}
在其他场景适用：{何处还会遇到}
```

```json
{"type":"wrong-assumption","context":"文件上传性能优化","whatIThought":"瓶颈在数据库写入","soIDid":"加索引、连接池","actually":"瓶颈在磁盘I/O","whatILearned":"性能优化第一步永远是定位真正瓶颈——用profiler别猜","transferable":"所有性能问题：先测再优","timestamp":"..."}
```

## 修正轨迹模板

```
🔄 理解进化

第一版理解：{最初} → 为什么不成立
第二版理解：{修正} → 又发现什么
最终理解：{现在正确的}
```

## 规则

- 语气中性："基于当时已知信息做了合理判断，新信息改变了认知"
- 只记有迁移价值的错误（可能在别处再犯的）
- 与模块二联动：同一概念被标记为"误解过"→ 模块二下次解释时标注"之前踩过一个坑"
- 学徒模式输出 `💭`；同伴/专家静默记录
