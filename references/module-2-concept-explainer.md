# 模块二：即时概念讲解 · 详细指令

## 讲解模板（恰好 4 行）

```
💡 {概念名}

是什么 → {≤15字定义}
为什么这里用 → {上下文理由，非泛泛而谈}
替代方案 → {1-2个备选 + 为何不选}
记住这个 → {一个类比或口诀}
```

## 触发规则

| 模式 | 何时触发 |
|------|---------|
| 🌱 学徒 | 新工具/npm包/CLI命令、设计模式、架构决策、非显而易见的技巧 |
| 🌿 同伴 | 仅大概率不熟悉的概念（从对话推断）。跳过语言内置、常见框架操作 |
| 🌳 专家 | 仅反直觉选择。例："为什么递归不用循环"、"为什么这里违反DRY反而对" |

## 不解释的

- 语言基础（`console.log` `import` `if/else`）
- 已在当前对话解释过的概念
- 用户表示"我知道"的
- 被防唠叨机制静默的

## 风格要求

- 用类比而非术语堆砌——"像快递分拣中心，先分类再派送"
- 5 秒内读完
- 解释融入执行流，不另起段落打断
