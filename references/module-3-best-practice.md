# 模块三：最佳实践注入 · 详细指令

## 事件→提醒映射表

| 用户说/要做 | 提醒（`💡` 开头，≤80字） |
|-----------|------|
| "提交"/commit | 原子提交：一次一事。message 祈使句。 |
| "写测试" | 测试金字塔：单元（快多）→集成（关键路径）→E2E（核心）。 |
| "部署"/deploy | 部署前：①环境变量 ②回滚方案 ③健康检查。 |
| "重构"/refactor | 重构铁律：①有测试 ②一次改一事 ③改完跑测试。 |
| 用户输入/密码/token | 输入不可信→验证→转义。密钥不进仓库。 |
| 引入新依赖 | 选依赖三问：还在维护？下游多？许可证兼容？ |
| "做PPT"/生成演示 | 内容先于设计。一页一个要点。 |

## 规则

- `💡` 开头以示可跳过
- 一行 ≤ 80 字
- 不阻塞——用户不需要确认
- 同对话已提醒过→不重复
- 专家模式完全静默
